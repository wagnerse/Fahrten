## Directory Purposes

**fahrtenplaner/:**
- Purpose: Main application package with all source code
- Contains: Python modules for web UI, data models, optimization logic, and external API integration
- Key files: app.py (entry point), models.py (domain objects)

**.streamlit/:**
- Purpose: Streamlit framework configuration
- Contains: config.toml with theme colors and server settings
- Key files: config.toml (defines primaryColor #1f77b4, headless mode true)

**.planning/codebase/:**
- Purpose: GSD codebase analysis documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md
- Auto-generated by orchestrator; serves as reference for future implementations

## Key File Locations

**Entry Points:**
- `fahrtenplaner/app.py`: Streamlit application – run with `streamlit run fahrtenplaner/app.py`
- `create_excel.py`: Excel generation utility (root level, legacy)

**Configuration:**
- `fahrtenplaner/.streamlit/config.toml`: Streamlit UI theme and server behavior
- `pyproject.toml`: Project metadata (name, version, dependencies)
- `requirements.txt`: Python package dependencies (streamlit, pandas, httpx, etc.)

**Core Logic:**
- `fahrtenplaner/optimizer.py`: Bitmask-DP optimization algorithm (lines 24-394, optimize_day function)
- `fahrtenplaner/models.py`: Tour, Connection, ChainLink, DayPlan dataclasses

**Data Access:**
- `fahrtenplaner/db_client.py`: DB transport.rest API client with rate limiting (lines 17-306)
- `fahrtenplaner/myres_client.py`: MyRES 3 login and tour fetching via curl (lines 144-370)

**Testing:**
- No test files present; testing is manual via Streamlit UI

## Naming Conventions

**Files:**
- `lowercase_with_underscores.py`: Python modules (app.py, db_client.py, models.py)
- `PascalCase.xlsx`: Excel data files (Freie_Touren_BB_MV_01-03_April_2026.xlsx)
- `UPPERCASE.md`: Documentation (README.md)
- `.streamlit/config.toml`: Configuration files in dotted directories

**Classes/Types:**
- `PascalCase`: All dataclasses and classes (Tour, Connection, ChainLink, DayPlan, Leg, MyRESClient, RateLimiter)

**Functions/Methods:**
- `snake_case`: All functions (optimize_day, lookup_station, find_connection, batch_lookup_stations, check_reachability_with_ids, load_tours_from_excel)
- `_private_snake_case`: Internal/private functions (_render_result, _render_tour_block, _check_transfer_warning, _parse_journey, _detect_columns, _row_to_tour)

**Variables/Properties:**
- `snake_case`: All variables and instance attributes (tours, home_station, departure_time, tour_nr, can_reach_from_home)
- `CONSTANT_CASE`: Constants (MIN_TRANSFER_MINUTES = 5, MAX_TRANSFER_GAP_HOURS = 12, BASE_URL = "https://v6.db.transport.rest")

**Booleans/Flags:**
- `is_replacement_service`: Boolean properties following is_/has_/can_ prefix pattern
- `same_station`: Boolean variable names

## Where to Add New Code

**New Feature (e.g., new filtering option):**
- Primary code: `fahrtenplaner/app.py` – add sidebar input and filtering logic (lines 74-319)
- Models impact: Update `fahrtenplaner/models.py` if new Tour property needed
- Tests: Add manual test cases to Streamlit UI sidebar

**New Optimization Constraint (e.g., max tours per day):**
- Implementation: `fahrtenplaner/optimizer.py` – modify DP transition (lines 280-300) or Phase 1-3 pruning
- Logic: Update reachability checks (lines 161-227) or pruning logic (lines 126-158)
- Tests: Verify with demo Excel data in UI; check progress logs

**New External Data Source (e.g., different booking system):**
- Data Access: Create new client module `fahrtenplaner/[system]_client.py` parallel to myres_client.py (lines 144-370)
- Integration: Update app.py sidebar to show selection between MyRES and new source
- Format: Ensure output matches Tour dataclass (fahrtenplaner/models.py)

**New Report/Export Format:**
- Location: `fahrtenplaner/app.py` – add new tab or expander (lines 271-390)
- Data source: Access plan.chain (DayPlan.chain property) or plan.tours (computed property)
- Rendering: Use Streamlit components (st.dataframe, st.download_button, st.markdown)

**Utilities (e.g., helper functions):**
- Station matching: Add to `fahrtenplaner/db_client.py` (see stations_match function, lines 258-260)
- Parsing: Add to `fahrtenplaner/myres_client.py` (see _parse_time, _parse_date functions, lines 65-99)
- Model properties: Add @property methods to dataclasses in `fahrtenplaner/models.py` (see examples at lines 25-42, 72-101)

## Special Directories

**fahrtenplaner/__pycache__/:**
- Purpose: Compiled Python bytecode cache
- Generated: Automatically by Python interpreter
- Committed: No (in .gitignore)

**.streamlit/:**
- Purpose: Streamlit configuration and secrets
- Committed: Yes (config.toml is safe; secrets would go in .streamlit/secrets.toml which is ignored)
- Modification: Edit config.toml for theme or server settings; no user secrets stored here

**.venv/:**
- Purpose: Local Python virtual environment with installed packages
- Committed: No (in .gitignore)
- Setup: Run `python -m venv .venv` then `.venv/bin/pip install -r requirements.txt`

**.planning/:**
- Purpose: GSD phase planning documents generated by `/gsd` commands
- Committed: No (in .gitignore typically, but can be version-controlled)
- Purpose: Serves as reference for implementation and quality gates

## Configuration & Initialization

**Application Startup:**
1. `streamlit run fahrtenplaner/app.py`
2. Streamlit reads `.streamlit/config.toml` for theme and server settings
3. app.py runs top-to-bottom, initializing session state and rendering UI
4. Session state persists for each browser session until tab closes

**Environment Requirements:**
- Python 3.13+ (specified in pyproject.toml)
- Virtual environment with packages from requirements.txt installed
- Network access to db.transport.rest API (free, no authentication)
- Network access to res.ivv-berlin.de for MyRES (requires login credentials)

**Demo Data:**
- Freie_Touren_BB_MV_01-03_April_2026.xlsx in project root
- Parsed by load_tours_from_excel() when "Demo-Daten laden" button clicked
- Column detection via regex patterns in _detect_columns()
